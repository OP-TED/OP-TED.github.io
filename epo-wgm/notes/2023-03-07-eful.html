Working Group meeting 07/03/2023

Date: 07/03/2023
Participants: Natalie Muric, Laszlo Ketszeri
Model editor: Andreea Pasăre
Note editor: Jana Ahmad

Agenda

  • Continue discussing eFulfillment

    • To continue from: ConsolidatedShipment

  • A party should have an authorization. This needs to be modelled.

  • Know how to integrate Certificate

Discussion

WG discussed the DespatchAdvice Class

  • epo-ful:TransportMeans is connected to adms:Identifier (despatch identifier)

  • Note: The order and dispatch advice should be met.

  • epo-playedBy relation is added between epo-ful:TransportMeansOperator and foaf:Person

  • Despatch Roles class is created:

  • epo-ful:ShipmentAgreement object is created and epo-ful:Shipment is removed

  • epo-ful:specifiesConsignment relation is created between epo-ful:DespatchAdvice and epo-ful:Consignment

  • epo-ful:specifiesDespatcher relation is created between epo-ful:DespatchAdvice and epo-ful:Despatcher

  • epo-ful:refersToConsignment relation is created between epo-ful:DespatchAdvice and epo-ful:Consignment

  • Despatch Quantities class is created and discussed:
