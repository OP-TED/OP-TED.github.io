Working Group meeting

Date: 17/10/2024  
Participants: Natalie Muric
Model editor:   Achilles Dougalis
Note editor: Grzegorz Kostkowski

Agenda

  • Revisit the _ePO 5.0.0 eForms _GitHub project issues.

Discussions

  • The ePO 5.0.0 eForms ePO project in GitHub was renamed to eForm Requirements.

Issue #708

(that relates to discussion #656).

  • An example notice XML document was analyzed to verify if and what information that is relevant to the request is already included in a notice description. The notice was downloaded from the EU tenders website.

  • The working group identified the following information that needs to be made available for the requesters:

    • eForms SDK version

    • the ontology version

  • It was confirmed that the ontology definition contains the version number that can be used by the requesters:

  • The following related properties were identified in the example notice file:

  • Documentation of UBL-ContractNotice was inspected to learn more about the cbc:CustomizationID term that is used to describe a notice.

  • The outcome of the analysis was that cbc:customizationID property cannot be used without the cbc:UBLVersionID (both versions need to be specified).
