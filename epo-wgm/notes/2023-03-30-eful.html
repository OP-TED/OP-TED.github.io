Working Group meeting 30/03/2023

Date: 30/03/2023
Participants:, Natalie Muric, Peter Borresen
Model editor: Andreea Pasăre
Note editor: Jana Ahmad

Agenda

  • Discuss the eFulfilment model with respect to the CEN-TC-440-Peppol-Despatch advice model:

    • To continue from Delivery Term

Discussion

The WG compared the CEN-TC-440-Peppol-Despatch advice model with our eFulfilment model and made changes accordingly, as follows.

  • To cover Handling instructions: epo-ful:hasHandlingInstruction is added between at-voc-new:handling-instructionis and epo-ful:TransportHandlingUnit

  • Transport equipment, Handling unit hazardous indicator, Shipping marks are covered in eFulfillment model.

  • epo-ful:fullness-indicator code is deleted

  • The usefulness of the data type property epo-ful:isReturnable needs to be evaluated.

  • It was noted that Returnable Item need to be discussed

  • To cover Packaging type code: epo-ful:Packaging-type code is added with epo-ful:hasPackagingType relation to epo-ful:package.

  • Minimum temperature, Measurement dimension, Maximum temperature: are covered in eFulfilment.

  • Goods item:

    • Identifier for epo-ful:GoodsItem is covered

    • Hazardous risk indicator is covered.

  • epo-ful:isHazardousRisk is removed from epo-ful:TransportHandlingUnit

  • Quantity for Goods Iem:

    • epo-ful:hasTotalPackageQuantity is deleted.

    • Quantity is covered by epo-ful:hasTotalGoodsItemQuantity

  • Dispatch line note

    • It is covered by dct:description in epo-cat:Line

  • epo:hasNodespatcheableQuantity Relation is deleted

  • For Document reference

    • epo-ful:hasAssociatedDocument is added to epo-ful:DepatchLine

  • Item information: is covered.

  • Good item definition with additional information are added.

  • Certificate: is covered
