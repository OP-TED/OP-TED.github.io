Working Group meeting 02/03/2023

Date: 02/03/2023
Participants:, Natalie Muric, Peter Borresen, Christian Mondini
Model editor: Andreea PasÄƒre
Note editor: Jana Ahmad

Agenda

  • Discuss the eFulfilment model

  • Discuss shipping mark

  • Discuss the concept of ReceivedAdvice

Discussion

The attribute "epo-ful:hasShippingMark" is present in both "epo-ful:TransportHandlingUnit" and "epo-ord:DeliveryInformation", leading to duplicated information. The working group has decided to keep it as is for now and further discuss it in the future.

The implementation of ReceivedAdvice will be postponed until we finish with DespatchAdvice.

The WG compared the CEN-TC-440-Peppol-Despatch advice model with our eFulfilment model and made changes accordingly, as follows.

  • epo-ful:hasAllowanceChargeReason relation is added to epo-cat:ChargInformation

  • The property 'epo-ful:hasBaseAmount' is added to the 'MonetaryValue' class to enable the use of base amount in eFulfilment.

  • A new relationship, epo-ful:hasAttachmentDocument, has been created to fulfill the requirement for an External Reference.

    • It needs to be checked how the (cac:AdditionalDocumentReference) is modeled in the Order module.

  • epo-ful:hasCarrierConsingnmentId has been added to epo-ful:Consignment

  • epo-ful:hasSpecialServiceInstructions has been added

  • epo-ful:NetWeightMeasure and ChargeableWeightMeasure have been added

  • epo-ful:hasConsignmentDeclaredStatisticsValue has been added
